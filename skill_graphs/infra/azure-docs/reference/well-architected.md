# Azure Well-Architected Framework
The Azure Well-Architected Framework is a set of quality-driven tenets, architectural decision points, and review tools intended to help solution architects build a technical foundation for their workloads.

## Design workloads that achieve business value over time.
As solution architects, you want to build reliable, secure, and performant workloads that maximize the value of investment in Azure infrastructure. Start with the Pillars, and align your design choices with the principles. Then, build a strong foundation for your workload based on technical design areas. Finally, use review tools to assess your readiness in deploying to production.

## Featured update - Develop a multi-region disaster recovery plan
Disaster recovery is the set of strategies and processes used to restore critical workloads and data after major disruptions such as outages or natural disasters. These events surpass the capabilities of built-in cloud resiliency. A well-structured disaster recovery plan defines roles, procedures, and communication protocols to ensure business continuity, minimize downtime, and enable reliable recovery across multiple regions.
[Develop a multi-region disaster recovery plan](https://learn.microsoft.com/en-us/azure/well-architected/design-guides/disaster-recovery)
## Pillars
  * ![](https://learn.microsoft.com/en-us/azure/well-architected/_images/reliability.svg)
[Reliability](https://learn.microsoft.com/en-us/azure/well-architected/reliability/)
Ensures that the workload meets the uptime and recovery targets by building redundancy and resiliency at scale.
  * ![](https://learn.microsoft.com/en-us/azure/well-architected/_images/security.svg)
[Security](https://learn.microsoft.com/en-us/azure/well-architected/security/)
Protect the workload from attacks by maintaining confidentiality and data integrity.
  * ![](https://learn.microsoft.com/en-us/azure/well-architected/_images/cost-optimization.svg)
[Cost Optimization](https://learn.microsoft.com/en-us/azure/well-architected/cost-optimization/)
Adopt an optimization mindset by at organizational, architectural, and tactical levels to keep your spending within budget.
  * ![](https://learn.microsoft.com/en-us/azure/well-architected/_images/operational-excellence.svg)
[Operational Excellence](https://learn.microsoft.com/en-us/azure/well-architected/operational-excellence/)
Reduce issues in production by building holistic observability and automated systems.
  * ![](https://learn.microsoft.com/en-us/azure/well-architected/_images/performance-efficiency.svg)
[Performance Efficiency](https://learn.microsoft.com/en-us/azure/well-architected/performance-efficiency/)
Adjust to changes in demands placed on the workload by through horizontal scaling and testing changes before deploying to production.


## Workloads
  * [Artificial Intelligence (AI)](https://learn.microsoft.com/en-us/azure/well-architected/ai/)
Incorporate discriminative or generative AI models in your workload for predictive analysis, generating content, and other AI use cases.
  * [Software as a service (SaaS)](https://learn.microsoft.com/en-us/azure/well-architected/saas/)
Key insights for Startups and mature independent software vendors (ISVs) to build scalable, performant, reliable, and secure SaaS solutions.
  * [Oracle on IaaS](https://learn.microsoft.com/en-us/azure/well-architected/oracle-iaas/overview)
Host Oracle workloads on Azure, which includes not only Oracle Databases, but also applications such as Siebel, Peoplesoft, JD Edwards.
  * [SAP](https://learn.microsoft.com/en-us/azure/well-architected/sap/quick-links)
Evaluate, design, and optimize an SAP workload from premigration to operations.
  * [Mission-critical](https://learn.microsoft.com/en-us/azure/well-architected/mission-critical/)
Design and operationalize workloads that are expected to be always available and resilient to failures.
  * [Sustainability](https://learn.microsoft.com/en-us/azure/well-architected/sustainability/)
Plan your path forward, improve your sustainability posture, and create new business value while reducing your operational footprint.
  * [Azure VMware Solution](https://learn.microsoft.com/en-us/azure/well-architected/azure-vmware/)
Relocate legacy application virtual machines to Azure VMware Solution as a staging area for the first phase of your migration and modernization strategy.
  * [Azure Virtual Desktop](https://learn.microsoft.com/en-us/azure/well-architected/azure-virtual-desktop/)
Run Windows desktops and applications on Azure from any device and location.
