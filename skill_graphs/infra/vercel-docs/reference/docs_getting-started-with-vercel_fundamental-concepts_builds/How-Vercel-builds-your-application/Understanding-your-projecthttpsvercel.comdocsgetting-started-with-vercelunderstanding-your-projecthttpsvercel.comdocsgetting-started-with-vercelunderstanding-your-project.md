##  [Understanding your project](https://vercel.com/docs/getting-started-with-vercel#understanding-your-project)[](https://vercel.com/docs/getting-started-with-vercel#understanding-your-project)
Before running any commands, Vercel inspects your project to understand what it's working with. This inspection looks at your package files, configuration, and directory structure to detect which [framework](https://vercel.com/docs/frameworks) you're using.
Framework detection matters because different frameworks have different build requirements. For example, a Next.js application needs `next build`, but a plain static site might not need a build command at all. By detecting your framework automatically, Vercel can apply sensible defaults without requiring you to configure anything.
When Vercel recognizes your framework, it applies a preset that configures the [install command](https://vercel.com/docs/deployments/configure-a-build#install-command), [build command](https://vercel.com/docs/deployments/configure-a-build#build-command), and [output directory](https://vercel.com/docs/deployments/configure-a-build#output-directory). You can override any of these settings if your project has specific requirements, but most projects work with the defaults.
