##  [The build environment](https://vercel.com/docs/getting-started-with-vercel#the-build-environment)[](https://vercel.com/docs/getting-started-with-vercel#the-build-environment)
Each build runs in its own isolated virtual machine. Vercel provisions this environment on-demand, ensuring your build has dedicated resources and can't be affected by other builds running on the platform. The environment comes pre-configured with common [build tools and runtimes](https://vercel.com/docs/deployments/build-image), including Node.js, Python, Ruby, and Go, so most projects can build without any special setup.
The isolation also provides security. Your source code, environment variables, and build artifacts remain private to your build. Once the build completes, the environment is destroyed.
