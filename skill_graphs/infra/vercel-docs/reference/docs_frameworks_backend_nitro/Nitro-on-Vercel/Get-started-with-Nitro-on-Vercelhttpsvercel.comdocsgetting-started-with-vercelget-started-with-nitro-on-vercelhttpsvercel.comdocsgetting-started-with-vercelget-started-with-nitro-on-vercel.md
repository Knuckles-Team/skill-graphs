##  [Get started with Nitro on Vercel](https://vercel.com/docs/getting-started-with-vercel#get-started-with-nitro-on-vercel)[](https://vercel.com/docs/getting-started-with-vercel#get-started-with-nitro-on-vercel)
To get started with Nitro on Vercel, use the following Nitro template to deploy to Vercel with zero configuration:
[![](https://7nyt0uhk7sse4zvn.public.blob.vercel-storage.com/docs-assets/static/topics/icons/nitro.svg)Deploy Nitro to Vercel](https://vercel.com/templates/backend/nitro-starter)
[Deploy](https://vercel.com/new/clone?repository-url=https%3A%2F%2Fgithub.com%2Fvercel%2Fvercel%2Ftree%2Fmain%2Fexamples%2Fnitro&template=nitro)
Vercel deployments can [integrate with your git provider](https://vercel.com/docs/git) to [generate preview URLs](https://vercel.com/docs/deployments/environments#preview-environment-pre-production) for each pull request you make to your Nitro project.
###  [Get started with Vercel CLI](https://vercel.com/docs/getting-started-with-vercel#get-started-with-vercel-cli)[](https://vercel.com/docs/getting-started-with-vercel#get-started-with-vercel-cli)
Get started by initializing a new Nitro project using [Vercel CLI init command](https://vercel.com/docs/cli/init):
terminal
```
vc init nitro
```

This will clone the `nitro`.
