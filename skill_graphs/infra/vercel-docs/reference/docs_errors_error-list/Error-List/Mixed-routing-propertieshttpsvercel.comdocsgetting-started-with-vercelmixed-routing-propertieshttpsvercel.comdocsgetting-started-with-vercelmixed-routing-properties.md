##  [Mixed routing properties](https://vercel.com/docs/getting-started-with-vercel#mixed-routing-properties)[](https://vercel.com/docs/getting-started-with-vercel#mixed-routing-properties)
If you have [`rewrites`](https://vercel.com/docs/project-configuration/vercel-json#rewrites), [`redirects`](https://vercel.com/docs/project-configuration/vercel-json#redirects), [`headers`](https://vercel.com/docs/project-configuration/vercel-json#headers), [`cleanUrls`](https://vercel.com/docs/project-configuration/vercel-json#cleanurls) or [`trailingSlash`](https://vercel.com/docs/project-configuration/vercel-json#trailingslash) defined in your [configuration](https://vercel.com/docs/project-configuration) file, then [`routes`](https://vercel.com/docs/project-configuration/vercel-json#routes) cannot be defined.
This is a necessary limitation because `routes` is a lower-level primitive that contains all of the other types. Therefore, it cannot be merged safely with the new properties.
See the [Upgrading Routes](https://vercel.com/docs/project-configuration/vercel-json#upgrading-legacy-routes) section for examples of `routes` compared to the new properties.
