# Data Cache for Next.js
Last updated February 9, 2026
Data Cache is available in [Beta](https://vercel.com/docs/release-phases#beta) on [all plans](https://vercel.com/docs/plans)
Data cache is a specialized, granular cache introduced with Next.js 13 for storing [Next.js App Router](https://vercel.com/docs/frameworks/nextjs). When using `fetch` or `unstable_cache`, Vercel automatically scaffolds globally distributed infrastructure for you with no additional configuration.
  * Find out [how Data cache works](https://vercel.com/docs/routing#how-data-cache-works)
  * [When to use it](https://vercel.com/docs/routing#when-to-use-data-cache)
  * Get started with the [examples](https://vercel.com/docs/routing#using-data-cache)


For Next.js 15 and above, see [Runtime Cache](https://vercel.com/docs/runtime-cache) for the recommended caching approach. Data cache is for Next.js 14 and below.
