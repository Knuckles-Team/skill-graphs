##  [V](https://vercel.com/docs/glossary#v)[](https://vercel.com/docs/glossary#v)
###  [v0](https://vercel.com/docs/glossary#v0)[](https://vercel.com/docs/glossary#v0)
An AI-powered tool that converts natural language descriptions into React code and UI components, integrated with Vercel for deployment.
###  [Vercel Authentication](https://vercel.com/docs/glossary#vercel-authentication)[](https://vercel.com/docs/glossary#vercel-authentication)
A deployment protection method that restricts access to team members and authorized users with Vercel accounts.
###  [Vercel Blob](https://vercel.com/docs/glossary#vercel-blob)[](https://vercel.com/docs/glossary#vercel-blob)
Scalable object storage service for static assets like images, videos, and files, optimized for global content delivery.
###  [Vercel Firewall](https://vercel.com/docs/glossary#vercel-firewall)[](https://vercel.com/docs/glossary#vercel-firewall)
A multi-layered security system that protects applications from threats, including platform-wide DDoS protection and customizable WAF rules.
###  [Vercel Functions](https://vercel.com/docs/glossary#vercel-functions)[](https://vercel.com/docs/glossary#vercel-functions)
Serverless compute that allows you to run server-side code without managing servers. Functions automatically scale based on demand and offer enhanced concurrency through [Fluid Compute](https://vercel.com/docs/fluid-compute) for AI workloads and I/O-bound tasks.
###  [Vercel Sandbox](https://vercel.com/docs/glossary#vercel-sandbox)[](https://vercel.com/docs/glossary#vercel-sandbox)
An ephemeral compute primitive for safely running untrusted or user-generated code in isolated Linux VMs, designed for AI agents, code generation, and developer experimentation.
###  [Virtual Experience Score (VES)](https://vercel.com/docs/glossary#virtual-experience-score-ves)[](https://vercel.com/docs/glossary#virtual-experience-score-ves)
A predictive performance metric that anticipates the impact of changes on application performance before deployment.
