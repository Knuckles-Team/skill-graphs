## v1.9.0 (2021-12-31)[¶](https://docs.pydantic.dev/latest/changelog/#v190-2021-12-31)
Thank you to pydantic's sponsors:
### Highlights[¶](https://docs.pydantic.dev/latest/changelog/#highlights)
  * add Python 3.10 support,
  * [Discriminated unions](https://docs.pydantic.dev/usage/types/#discriminated-unions-aka-tagged-unions),
  * [`Config.smart_union` for better union logic](https://docs.pydantic.dev/usage/model_config/#smart-union),
  * Binaries for Macos M1 CPUs,
  * Complex types can be set via [nested environment variables](https://docs.pydantic.dev/usage/settings/#parsing-environment-variable-values), e.g. `foo___bar`,
  * add a dark mode to _pydantic_ documentation,
  * Add support for autocomplete in VS Code via `__dataclass_transform__`,
  * Add "exclude" as a field parameter so that it can be configured using model config,

### v1.9.0a1 (2021-12-18) Changes[¶](https://docs.pydantic.dev/latest/changelog/#v190a1-2021-12-18-changes)
  * Add support for `Decimal`-specific validation configurations in `Field()`, additionally to using `condecimal()`, to allow better support from editors and tooling,
  * Add `arm64` binaries suitable for MacOS with an M1 CPU to PyPI,
  * Fix issue where `None` was considered invalid when using a `Union` type containing `Any` or `object`,
  * When generating field schema, pass optional `field` argument (of type `pydantic.fields.ModelField`) to `__modify_schema__()` if present,
  * Fix issue when pydantic fail to parse `typing.ClassVar` string type annotation,
  * Mention Python >= 3.9.2 as an alternative to `typing_extensions.TypedDict`,
  * Changed the validator method name in the [Custom Errors example](https://docs.pydantic.dev/usage/models/#custom-errors) to more accurately describe what the validator is doing; changed from `name_must_contain_space` to `value_must_equal_bar`,
  * Add `AmqpDsn` class,
  * Always use `Enum` value as default in generated JSON schema,
  * Add support for Mypy 0.920,
  * `validate_arguments` now supports `extra` customization (used to always be `Extra.forbid`),
  * Complex types can be set by nested environment variables,
  * Fix mypy plugin to collect fields based on `pydantic.utils.is_valid_field` so that it ignores untyped private variables,
  * fix `validate_arguments` issue with `Config.validate_all`,
  * avoid dict coercion when using dict subclasses as field type,
  * add support for `object` type,
  * Updates pydantic dataclasses to keep `_special` properties on parent classes,
  * Add a `TypedDict` class for error objects,
  * Fix support for using a subclass of an annotation as a default,
  * make `create_model_from_typeddict` mypy compliant,
  * Make multiple inheritance work when using `PrivateAttr`,
  * Parse environment variables as JSON, if they have a `Union` type with a complex subfield,
  * Prevent `StrictStr` permitting `Enum` values where the enum inherits from `str`,
  * Make `SecretsSettingsSource` parse values being assigned to fields of complex types when sourced from a secrets file, just as when sourced from environment variables,
  * add a dark mode to _pydantic_ documentation,
  * Make `pydantic-mypy` plugin compatible with `pyproject.toml` configuration, consistent with `mypy` changes. See the [doc](https://docs.pydantic.dev/mypy_plugin/#configuring-the-plugin) for more information,
  * add Python 3.10 support,
  * Correctly parse generic models with `Json[T]`,
  * Update contrib docs re: Python version to use for building docs,
  * Clarify documentation about _pydantic_ 's support for custom validation and strict type checking, despite _pydantic_ being primarily a parsing library,
  * Fix schema generation for `Deque` fields,
  * fix an edge case when mixing constraints and `Literal`,
  * Fix postponed annotation resolution for `NamedTuple` and `TypedDict` when they're used directly as the type of fields within Pydantic models,
  * Fix bug when `mypy` plugin fails on `construct` method call for `BaseSettings` derived classes,
  * Add function overloading for a `pydantic.create_model` function,
  * Fix mypy plugin issue with self field declaration,
  * The colon at the end of the line "The fields which were supplied when user was initialised:" suggests that the code following it is related. Changed it to a period,
  * Renamed variable `schema` to `schema_` to avoid shadowing of global variable name,
  * Add support for autocomplete in VS Code via `__dataclass_transform__`,
  * add missing type annotations in `BaseConfig` and handle `max_length = 0`,
  * Change `orm_mode` checking to allow recursive ORM mode parsing with dicts,
  * Add episode 313 of the _Talk Python To Me_ podcast, where Michael Kennedy and Samuel Colvin discuss Pydantic, to the docs,
  * fix JSON schema generation when a field is of type `NamedTuple` and has a default value,
  * `Enum` fields now properly support extra kwargs in schema generation,
  * **Breaking Change, see** : Make serialization of referenced pydantic models possible,
  * Add `uniqueItems` option to `ConstrainedList`,
  * Try to evaluate forward refs automatically at model creation,
  * Switch docs preview and coverage display to use
  * Add `__version__` attribute to pydantic module,
  * Add `postgresql+asyncpg`, `postgresql+pg8000`, `postgresql+psycopg2`, `postgresql+psycopg2cffi`, `postgresql+py-postgresql` and `postgresql+pygresql` schemes for `PostgresDsn`,
  * Enable the Hypothesis plugin to generate a constrained decimal when the `decimal_places` argument is specified,
  * Allow `collections.abc.Callable` to be used as type in Python 3.9,
  * Documentation update how to custom compile pydantic when using pip install, small change in `setup.py` to allow for custom CFLAGS when compiling,
  * remove side effect of `default_factory` to run it only once even if `Config.validate_all` is set,
  * Add lookahead to ip regexes for `AnyUrl` hosts. This allows urls with DNS labels looking like IPs to validate as they are perfectly valid host names,
  * Set `minItems` and `maxItems` in generated JSON schema for fixed-length tuples,
  * Add `strict` argument to `conbytes`,
  * Support user defined generic field types in generic models,
  * Add an example and a short explanation of subclassing `GetterDict` to docs,
  * add `KafkaDsn` type, `HttpUrl` now has default port 80 for http and 443 for https,
  * Add `PastDate` and `FutureDate` types,
  * Support generating schema for `Generic` fields with subtypes,
  * fix(encoder): serialize `NameEmail` to str,
  * add `Config.smart_union` to prevent coercion in `Union` if possible, see [the doc](https://docs.pydantic.dev/usage/model_config/#smart-union) for more information,
  * Add ability to use `typing.Counter` as a model field type,
  * Add parameterised subclasses to `__bases__` when constructing new parameterised classes, so that `A <: B => A[int] <: B[int]`,
  * Create `FileUrl` type that allows URLs that conform to `host_required` parameter, which is `True` by default (`AnyUrl` and subclasses), `False` in `RedisDsn`, `FileUrl`,
  * add `confrozenset()`, analogous to `conset()` and `conlist()`,
  * stop calling parent class `root_validator` if overridden,
  * Add `repr` (defaults to `True`) parameter to `Field`, to hide it from the default representation of the `BaseModel`,
  * Accept empty query/fragment URL parts,
