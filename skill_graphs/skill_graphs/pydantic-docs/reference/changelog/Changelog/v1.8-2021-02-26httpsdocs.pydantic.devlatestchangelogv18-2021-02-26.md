## v1.8 (2021-02-26)[¶](https://docs.pydantic.dev/latest/changelog/#v18-2021-02-26)
Thank you to pydantic's sponsors:
### Highlights[¶](https://docs.pydantic.dev/latest/changelog/#highlights_1)
  * [Hypothesis plugin](https://docs.pydantic.dev/hypothesis_plugin/) for testing,
  * support for [`NamedTuple` and `TypedDict`](https://docs.pydantic.dev/usage/types/#annotated-types),
  * Support [`Annotated` hints on model fields](https://docs.pydantic.dev/usage/schema/#typingannotated-fields),
  * [`frozen` parameter on `Config`](https://docs.pydantic.dev/usage/model_config/) to allow models to be hashed,


### Changes[¶](https://docs.pydantic.dev/latest/changelog/#changes_13)
  * **Breaking Change** , remove old deprecation aliases from v1,
    * remove notes on migrating to v1 in docs
    * remove `Schema` which was replaced by `Field`
    * remove `Config.case_insensitive` which was replaced by `Config.case_sensitive` (default `False`)
    * remove `Config.allow_population_by_alias` which was replaced by `Config.allow_population_by_field_name`
    * remove `model.fields` which was replaced by `model.__fields__`
    * remove `model.to_string()` which was replaced by `str(model)`
    * remove `model.__values__` which was replaced by `model.__dict__`
  * **Breaking Change:** always validate only first sublevel items with `each_item`. There were indeed some edge cases with some compound types where the validated items were the last sublevel ones,
  * Update docs extensions to fix local syntax highlighting,
  * fix: allow `utils.lenient_issubclass` to handle `typing.GenericAlias` objects like `list[str]` in Python >= 3.9,
  * Improve field declaration for _pydantic_ `dataclass` by allowing the usage of _pydantic_ `Field` or `'metadata'` kwarg of `dataclasses.field`,
  * Making `typing-extensions` a required dependency,
  * Make `resolve_annotations` more lenient, allowing for missing modules,
  * Allow configuring models through class kwargs,
  * Prevent `Mapping` subclasses from always being coerced to `dict`,
  * fix: allow `None` for type `Optional[conset / conlist]`,
  * Support empty tuple type,
  * fix: `python_requires` metadata to require >=3.6.1,
  * Properly encode `Decimal` with, or without any decimal places,
  * fix: update `__fields_set__` in `BaseModel.copy(update=…)`,
  * fix: keep order of fields with `BaseModel.construct()`,
  * Support generating schema for Generic fields,
  * Fix `validate_decorator` so `**kwargs` doesn't exclude values when the keyword has the same name as the `*args` or `**kwargs` names,
  * Prevent overriding positional arguments with keyword arguments in `validate_arguments`, as per behaviour with native functions,
  * add documentation for `con*` type functions,
  * Support custom root type (aka `__root__`) when using `parse_obj()` with nested models,
  * Support custom root type (aka `__root__`) with `from_orm()`,
  * ensure cythonized functions are left untouched when creating models, based on
  * Resolve forward refs for stdlib dataclasses converted into _pydantic_ ones,
  * Add support for `NamedTuple` and `TypedDict` types. Those two types are now handled and validated when used inside `BaseModel` or _pydantic_ `dataclass`. Two utils are also added `create_model_from_namedtuple` and `create_model_from_typeddict`,
  * Do not ignore annotated fields when type is `Union[Type[...], ...]`,
  * Raise a user-friendly `TypeError` when a `root_validator` does not return a `dict` (e.g. `None`),
  * Add a `FrozenSet[str]` type annotation to the `allowed_schemes` argument on the `strict_url` field type,
  * add `allow_mutation` constraint to `Field`,
  * Allow `Field` with a `default_factory` to be used as an argument to a function decorated with `validate_arguments`,
  * Allow non-existent secrets directory by only issuing a warning,
  * fix URL regex to parse fragment without query string,
  * fix: ensure to always return one of the values in `Literal` field type,
  * Support `typing.Annotated` hints on model fields. A `Field` may now be set in the type hint with `Annotated[..., Field(...)`; all other annotations are ignored but still visible with `get_type_hints(..., include_extras=True)`,
  * Added `StrictBytes` type as well as `strict=False` option to `ConstrainedBytes`,
  * added `Config.anystr_lower` and `to_lower` kwarg to `constr` and `conbytes`,
  * Support plain `typing.Tuple` type,
  * Add a bound method `validate` to functions decorated with `validate_arguments` to validate parameters without actually calling the function,
  * Add the ability to customize settings sources (add / disable / change priority order),
  * Fix mypy complaints about most custom _pydantic_ types,
  * Add a [usage details here](https://docs.pydantic.dev/hypothesis_plugin/),
  * add validator for `None`, `NoneType` or `Literal[None]`,
  * Handle properly fields of type `Callable` with a default value,
  * Updated `create_model` return type annotation to return type which inherits from `__base__` argument,
  * Add merged `json_encoders` inheritance,
  * allow overwriting `ClassVar`s in sub-models without having to re-annotate them,
  * add default encoder for `Pattern` type,
  * Add `NonNegativeInt`, `NonPositiveInt`, `NonNegativeFloat`, `NonPositiveFloat`,
  * Use % for percentage in string format of colors,
  * Fixed issue causing `KeyError` to be raised when building schema from multiple `BaseModel` with the same names declared in separate classes,
  * Add `rediss` (Redis over SSL) protocol to `RedisDsn` Allow URLs without `user` part (e.g., `rediss://:pass@localhost`),
  * Add a new `frozen` boolean parameter to `Config` (default: `False`). Setting `frozen=True` does everything that `allow_mutation=False` does, and also generates a `__hash__()` method for the model. This makes instances of the model potentially hashable if all the attributes are hashable,
  * fix schema generation with multiple Enums having the same name,
  * Added support for 13/19 digits VISA credit cards in `PaymentCardNumber` type,
  * fix: prevent `RecursionError` while using recursive `GenericModel`s,
  * use `enum` for `typing.Literal` in JSON schema,
  * Fix: some recursive models did not require `update_forward_refs` and silently behaved incorrectly,
  * Fix bug where generic models with fields where the typevar is nested in another type `a: List[T]` are considered to be concrete. This allows these models to be subclassed and composed as expected,
  * Add `Config.copy_on_model_validation` flag. When set to `False`, _pydantic_ will keep models used as fields untouched on validation instead of reconstructing (copying) them,
