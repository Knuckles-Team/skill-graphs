## v1.9.1 (2022-05-19)[¶](https://docs.pydantic.dev/latest/changelog/#v191-2022-05-19)
Thank you to pydantic's sponsors:
  * Limit the size of `generics._generic_types_cache` and `generics._assigned_parameters` to avoid unlimited increase in memory usage,
  * Add Jupyverse and FPS as Jupyter projects using pydantic,
  * Speedup `__isinstancecheck__` on pydantic models when the type is not a model, may also avoid memory "leaks",
  * Fix in-place modification of `FieldInfo` that caused problems with PEP 593 type aliases,
  * Add support for autocomplete in VS Code via `__dataclass_transform__` when using `pydantic.dataclasses.dataclass`,
  * Remove benchmarks from codebase and docs,
  * Typing checking with pyright in CI, improve docs on vscode/pylance/pyright,
  * Fix nested Python dataclass schema regression,
  * Update documentation about lazy evaluation of sources for Settings,
  * Prevent subclasses of bytes being converted to bytes,
  * Fixed "error checking inheritance of" when using PEP585 and PEP604 type hints,
  * Allow self referencing `ClassVar`s in models,
  * **Breaking Change, see** : Fix issue with self-referencing dataclass,
  * Include non-standard port numbers in rendered URLs,
  * `Config.copy_on_model_validation` does a deep copy and not a shallow one,
  * fix: clarify that discriminated unions do not support singletons,
  * Add `read_text(encoding='utf-8')` for `setup.py`,
  * Fix JSON Schema generation for Discriminated Unions within lists,
