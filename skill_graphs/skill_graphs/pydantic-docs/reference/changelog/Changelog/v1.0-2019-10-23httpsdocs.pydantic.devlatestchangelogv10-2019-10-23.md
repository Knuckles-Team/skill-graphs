## v1.0 (2019-10-23)[¶](https://docs.pydantic.dev/latest/changelog/#v10-2019-10-23)
  * **Breaking Change:** deprecate the `Model.fields` property, use `Model.__fields__` instead,
  * **Breaking Change:** Change the precedence of aliases so child model aliases override parent aliases, including using `alias_generator`,
  * **Breaking change:** Rename `skip_defaults` to `exclude_unset`, and add ability to exclude actual defaults,
  * Add `**kwargs` to `pydantic.main.ModelMetaclass.__new__` so `__init_subclass__` can take custom parameters on extended `BaseModel` classes,
  * Fix field of a type that has a default value,
  * Use `FutureWarning` instead of `DeprecationWarning` when `alias` instead of `env` is used for settings models,
  * Fix issue with `BaseSettings` inheritance and `alias` getting set to `None`,
  * Modify `__repr__` and `__str__` methods to be consistent across all public classes, add `__pretty__` to support python-devtools,
  * deprecation warning for `case_insensitive` on `BaseSettings` config,
  * For `BaseSettings` merge environment variables and in-code values recursively, as long as they create a valid object when merged together, to allow splitting init arguments,
  * change secret types example,
  * Change the signature of `Model.construct()` to be more user-friendly, document `construct()` usage,
  * Add example for the `construct()` method,
  * Improve use of `Field` constraints on complex types, raise an error if constraints are not enforceable, also support tuples with an ellipsis `Tuple[X, ...]`, `Sequence` and `FrozenSet` in schema,
  * update docs for bool missing valid value,
  * Better `str`/`repr` logic for `ModelField`,
  * Fix `ConstrainedList`, update schema generation to reflect `min_items` and `max_items` `Field()` arguments,
  * Allow abstracts sets (eg. dict keys) in the `include` and `exclude` arguments of `dict()`,
  * Fix JSON serialization errors on `ValidationError.json()` by using `pydantic_encoder`,
  * Clarify usage of `remove_untouched`, improve error message for types with no validators,
