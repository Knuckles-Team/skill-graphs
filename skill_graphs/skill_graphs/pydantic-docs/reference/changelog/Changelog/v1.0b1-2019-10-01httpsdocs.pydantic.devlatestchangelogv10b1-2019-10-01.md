## v1.0b1 (2019-10-01)[¶](https://docs.pydantic.dev/latest/changelog/#v10b1-2019-10-01)
  * **Breaking Change:** rename `Schema` to `Field`, make it a function to placate mypy,
  * **Breaking Change:** modify parsing behavior for `bool`,
  * **Breaking Change:** `get_validators` is no longer recognised, use `__get_validators__`. `Config.ignore_extra` and `Config.allow_extra` are no longer recognised, use `Config.extra`,
  * **Breaking Change:** modify default config settings for `BaseSettings`; `case_insensitive` renamed to `case_sensitive`, default changed to `case_sensitive = False`, `env_prefix` default changed to `''` - e.g. no prefix,
  * **Breaking change:** Implement `root_validator` and rename root errors from `__obj__` to `__root__`,
  * **Breaking Change:** alter the behaviour of `dict(model)` so that sub-models are nolonger converted to dictionaries,
  * **Breaking change:** Added `initvars` support to `post_init_post_parse`,
  * **Breaking Change:** Make `BaseModel.json()` only serialize the `__root__` key for models with custom root,
  * **Breaking Change:** complete rewrite of `URL` parsing logic,
  * **Breaking Change:** preserve superclass annotations for field-determination when not provided in subclass,
  * **Breaking Change:** `BaseSettings` now uses the special `env` settings to define which environment variables to read, not aliases,
  * add support for `assert` statements inside validators,
  * Update documentation to specify the use of `pydantic.dataclasses.dataclass` and subclassing `pydantic.BaseModel`,
  * Allow custom JSON decoding and encoding via `json_loads` and `json_dumps` `Config` properties,
  * make all annotated fields occur in the order declared,
  * use pytest to test `mypy` integration,
  * add `__repr__` method to `ErrorWrapper`,
  * Added support for `FrozenSet` members in dataclasses, and a better error when attempting to use types from the `typing` module that are not supported by Pydantic,
  * add documentation for Pycharm Plugin,
  * fix broken examples in the docs,
  * moving typing related objects into `pydantic.typing`,
  * Minor performance improvements to `ErrorWrapper`, `ValidationError` and datetime parsing,
  * Improvements to `datetime`/`date`/`time`/`timedelta` types: more descriptive errors, change errors to `value_error` not `type_error`, support bytes,
  * fix error messages for `Literal` types with multiple allowed values,
  * Improved auto-generated `title` field in JSON schema by converting underscore to space,
  * support `mypy --no-implicit-reexport` for dataclasses, also respect `--no-implicit-reexport` in pydantic itself,
  * add the `PaymentCardNumber` type,
  * Fix const validations for lists,
  * Set `additionalProperties` to false in schema for models with extra fields disallowed,
  * `EmailStr` validation method now returns local part case-sensitive per RFC 5321,
  * Added ability to validate strictness to `ConstrainedFloat`, `ConstrainedInt` and `ConstrainedStr` and added `StrictFloat` and `StrictInt` classes,
  * Improve handling of `None` and `Optional`, replace `whole` with `each_item` (inverse meaning, default `False`) on validators,
  * add support for `Type[T]` type hints,
  * Performance improvements from removing `change_exceptions`, change how pydantic error are constructed,
  * Fix the error message arising when a `BaseModel`-type model field causes a `ValidationError` during parsing,
  * allow `getter_dict` on `Config`, modify `GetterDict` to be more like a `Mapping` object and thus easier to work with,
  * Only check `TypeVar` param on base `GenericModel` class,
  * rename `Model._schema_cache` -> `Model.__schema_cache__`, `Model._json_encoder` -> `Model.__json_encoder__`, `Model._custom_root_type` -> `Model.__custom_root_type__`,
