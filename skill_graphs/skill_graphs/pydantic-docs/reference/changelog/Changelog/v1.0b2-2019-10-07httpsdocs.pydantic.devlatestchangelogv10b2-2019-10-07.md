## v1.0b2 (2019-10-07)[¶](https://docs.pydantic.dev/latest/changelog/#v10b2-2019-10-07)
  * Mark `StrictBool` typecheck as `bool` to allow for default values without mypy errors,
  * Transfer the documentation build from sphinx to mkdocs, re-write much of the documentation,
  * Add support for custom naming schemes for `GenericModel` subclasses,
  * Add `if TYPE_CHECKING:` to the excluded lines for test coverage,
  * Rename `allow_population_by_alias` to `allow_population_by_field_name`, remove unnecessary warning about it,
