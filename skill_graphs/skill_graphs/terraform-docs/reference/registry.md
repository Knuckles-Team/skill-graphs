# Publish to the Terraform registry
Learn how to publish Terraform providers, modules, and policies to the public Terraform registry.
The
![screenshot: terraform registry landing page](https://web-unified-docs-hashicorp.vercel.app/api/assets/terraform-docs-common/latest/img/docs/registry1.png)
The Terraform Registry is integrated [directly into Terraform](https://developer.hashicorp.com/terraform/language/providers/requirements) so you can directly specify providers and modules. Anyone can publish and consume providers, modules, and policies on the public [private registry](https://developer.hashicorp.com/terraform/registry/private) or [reference repositories and other sources directly](https://developer.hashicorp.com/terraform/language/modules/sources).
Use the navigation to the left to learn more about using the Terraform Registry.
##  [](https://developer.hashicorp.com/terraform/registry#navigating-the-registry)Navigating the Registry
The registry has a number of different categories for modules, providers, and policies to help with navigating the large number of available options. Select a provider or module card to learn more, filter results to a [specific tier](https://developer.hashicorp.com/terraform/registry/providers#provider-tiers-amp-namespaces), or use the search field at the top of the Registry. Search supports keyboard navigation.
![screenshot: terraform registry browse](https://web-unified-docs-hashicorp.vercel.app/api/assets/terraform-docs-common/latest/img/docs/registry2.png)
##  [](https://developer.hashicorp.com/terraform/registry#user-account)User Account
Individuals interested in publishing can use a GitHub account to sign in to the Terraform Registry. Click the **Sign-in** button, and follow the login prompts to authorize the use of your GitHub account and sign in. Refer to the specific instructions for publishing [modules](https://developer.hashicorp.com/terraform/registry/modules/publish), [providers](https://developer.hashicorp.com/terraform/registry/providers/publishing), and [policy libraries](https://developer.hashicorp.com/terraform/registry/policy-libraries/publishing).
![screenshot: terraform registry sign in](https://web-unified-docs-hashicorp.vercel.app/api/assets/terraform-docs-common/latest/img/docs/user-account.png)
If you are an HCP Terraform user and want to publish and manage modules or providers in the public registry, you can connect a GitHub account to an HCP Terraform organization, letting members collectively manage that account’s public registry artifacts. Refer to [Namespaces](https://developer.hashicorp.com/terraform/cloud-docs/public-namespace) for more information.
##  [](https://developer.hashicorp.com/terraform/registry#getting-help)Getting Help
The providers, modules, and policies available on the Terraform Registry are maintained either directly by HashiCorp, by trusted HashiCorp partners, or members of the Terraform community Refer to [Provider Tiers and Namespaces](https://developer.hashicorp.com/terraform/registry/providers#provider-tiers-amp-namespaces) for details. To create issues and contributions in GitHub, click the **Report an issue** link on the provider, module, or policy library's details page:
![Provider report issue link](https://web-unified-docs-hashicorp.vercel.app/api/assets/terraform-docs-common/latest/img/docs/registry-issue.png)
If you are having trouble publishing new content to the Terraform Registry, [refer to the Publishing FAQ](https://developer.hashicorp.com/terraform/registry/faq) for answers to some of the most commonly-encountered problems.
