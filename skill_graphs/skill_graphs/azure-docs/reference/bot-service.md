## Development options
There's more than one way to build and deploy a chatbot. Let's take a look at some options.
![](https://learn.microsoft.com/en-us/azure/bot-service/media/logos/logo-bot-sdk.svg?view=azure-bot-service-4.0) [Bot Framework SDK](https://learn.microsoft.com/en-us/azure/bot-service/bot-service-overview-introduction?view=azure-bot-service-4.0)
A modular set of developer tools to build, test, deploy, and manage chatbots in C# or JavaScript.
![](https://learn.microsoft.com/en-us/azure/bot-service/media/logos/logo-copilot-studio.svg?view=azure-bot-service-4.0) [Copilot Studio](https://learn.microsoft.com/en-us/microsoft-copilot-studio/fundamentals-what-is-copilot-studio)
Create agents with an easy to use graphical interface. No coding or AI expertise required. Works with Power Apps, and extendable with code.
![](https://learn.microsoft.com/en-us/azure/bot-service/media/logos/logo-composer.svg?view=azure-bot-service-4.0) [Bot Framework Composer](https://learn.microsoft.com/en-us/composer/introduction)
An open-source IDE built on the Bot Framework SDK. Composer lets you build chatbots with a visual authoring canvas, and extend with code.
