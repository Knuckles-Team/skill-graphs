## Availability and disaster recovery
Expand table
New feature or update | Details
---|---
**Always On availability groups** |
[Availability group asynchronous page request dispatching improvement](https://learn.microsoft.com/en-us/sql/database-engine/availability-groups/windows/failover-and-failover-modes-always-on-availability-groups?view=sql-server-ver17#asynchronous-page-request-dispatching-improvement) | Perform asynchronous page requests and in batches during failover recovery. Enabled by default.
[Allow database to switch to resolving state](https://learn.microsoft.com/en-us/sql/database-engine/availability-groups/windows/failover-and-failover-modes-always-on-availability-groups?view=sql-server-ver17#databases-switch-to-resolving-state-after-a-failure) | After a failure to read the persisted configuration data due to network service interruption.
[Configure AG group commit wait in milliseconds](https://learn.microsoft.com/en-us/sql/database-engine/configure-windows/availability-group-commit-time-server-configuration-options?view=sql-server-ver17) | Set `availability group commit time` in milliseconds for an availability group replica so that transactions are sent to the secondary replica faster.
[Control communication flow for availability groups](https://learn.microsoft.com/en-us/sql/database-engine/configure-windows/ucs-flow-control-sp-configure?view=sql-server-ver17) | A new `sp_configure` option lets the primary replica determine if the secondary replica falls behind. With the new configuration option, you can optimize communication between HADR endpoints.
[Distributed AG support for a contained AG](https://learn.microsoft.com/en-us/sql/database-engine/availability-groups/windows/contained-availability-groups-overview?view=sql-server-ver17#distributed-availability-groups) | Configure a distributed availability group between two contained availability groups.
[Distributed AG synchronization improvements](https://learn.microsoft.com/en-us/sql/database-engine/availability-groups/windows/distributed-availability-groups?view=sql-server-ver17#improvement-to-distributed-ag-synchronization) | Improves synchronization performance by reducing network saturation when the global primary and forwarder replicas are in asynchronous commit mode.
Fast failover for [persistent AG health issues](https://learn.microsoft.com/en-us/sql/database-engine/availability-groups/windows/failover-and-failover-modes-always-on-availability-groups?view=sql-server-ver17#fast-failover-for-persistent-health-issues) | Set the `RestartThreshold` value for an Always On availability group to 0, which tells the Windows Server Failover Cluster (WSFC) to fail over the availability group resource immediately when a persistent health issue is detected.
[Improved health check timeout diagnostics](https://learn.microsoft.com/en-us/sql/database-engine/availability-groups/windows/availability-group-lease-healthcheck-timeout?view=sql-server-ver17#sql-server-2025-improved-health-check-timeout-diagnostics) | Improves synchronization performance by reducing network saturation when the global primary and forwarder replicas are in asynchronous commit mode. This change is enabled by default and doesn't require any configuration.
[REMOVE listener IP address](https://learn.microsoft.com/en-us/sql/t-sql/statements/alter-availability-group-transact-sql?view=sql-server-ver17#remove-ip--four_part_ipv4_address--ipv6_address-) | New parameter in the `ALTER AVAILABILITY GROUP` Transact-SQL command lets you remove an IP address from a listener without deleting the listener.
Set `NONE` for [read-only](https://learn.microsoft.com/en-us/sql/t-sql/statements/alter-availability-group-transact-sql?view=sql-server-ver17#read_only_routing_url-tcpsystem-addressport--none) or [read-write](https://learn.microsoft.com/en-us/sql/t-sql/statements/alter-availability-group-transact-sql?view=sql-server-ver17#read_write_routing_url--tcpsystem-addressport--none) routing | When configuring `READ_WRITE_ROUTING_URL` and `READ_ONLY_ROUTING_URL`, you can set to `NONE` to revert specified routing by using the `ALTER AVAILABILITY GROUP` Transact-SQL command to automatically route traffic back to the primary replica.
[Configure TLS 1.3 encryption with TDS 8.0](https://learn.microsoft.com/en-us/sql/relational-databases/security/networking/connect-with-strict-encryption?view=sql-server-ver17#connect-to-an-always-on-availability-group) | Configure [TLS 1.3](https://learn.microsoft.com/en-us/sql/relational-databases/security/networking/tls-1-3?view=sql-server-ver17) encryption for communication between WSFC and an Always On availability group replica with [TDS 8.0](https://learn.microsoft.com/en-us/sql/relational-databases/security/networking/tds-8?view=sql-server-ver17) support.
**Always On failover cluster instance** |
[Configure TLS 1.3 encryption with TDS 8.0](https://learn.microsoft.com/en-us/sql/relational-databases/security/networking/connect-with-strict-encryption?view=sql-server-ver17#connect-to-a-failover-cluster-instance) | Configure [TLS 1.3](https://learn.microsoft.com/en-us/sql/relational-databases/security/networking/tls-1-3?view=sql-server-ver17) encryption for communication between WSFC and Always On failover cluster instance (FCI) with [TDS 8.0](https://learn.microsoft.com/en-us/sql/relational-databases/security/networking/tds-8?view=sql-server-ver17) support.
**Backups** |
[Back up to immutable blob storage](https://learn.microsoft.com/en-us/sql/relational-databases/backup-restore/sql-server-backup-to-url?view=sql-server-ver17#azure-immutable-storage-support) | Available when backing up to URL.
[Back up on secondary replicas](https://learn.microsoft.com/en-us/sql/database-engine/availability-groups/windows/active-secondaries-backup-on-secondary-replicas-always-on-availability-groups?view=sql-server-ver17#new-for-sql-server-2025) | In addition to copy-only backups, you can now also perform full and differential backups on any secondary replica.

[Configure TLS 1.3 encryption with TDS 8.0](https://learn.microsoft.com/en-us/sql/database-engine/log-shipping/about-log-shipping-sql-server?view=sql-server-ver17#enforce-tls-13-encryption) | Configure [TLS 1.3](https://learn.microsoft.com/en-us/sql/relational-databases/security/networking/tls-1-3?view=sql-server-ver17) encryption for communication between servers in a log shipping topology.
