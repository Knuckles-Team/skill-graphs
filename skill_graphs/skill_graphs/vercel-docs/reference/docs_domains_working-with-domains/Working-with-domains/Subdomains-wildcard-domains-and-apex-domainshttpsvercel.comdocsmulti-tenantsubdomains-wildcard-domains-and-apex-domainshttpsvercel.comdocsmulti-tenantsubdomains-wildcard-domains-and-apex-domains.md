##  [Subdomains, wildcard domains, and apex domains](https://vercel.com/docs/multi-tenant#subdomains-wildcard-domains-and-apex-domains)[](https://vercel.com/docs/multi-tenant#subdomains-wildcard-domains-and-apex-domains)
###  [Apex Domain](https://vercel.com/docs/multi-tenant#apex-domain)[](https://vercel.com/docs/multi-tenant#apex-domain)
The apex domain is the root-level domain, such as `acme.com`. When you add an apex domain, Vercel will recommend that you add a [redirect](https://vercel.com/docs/domains/deploying-and-redirecting#redirecting-www-domains) to a `www` subdomain. This is because `www` records allow for better control over your domain. Anything configured on the apex domain (for example, cookies or CAA records), will usually apply to all subdomains, rather than setting it on the `www` subdomain, which will only apply to your `www` record. In addition, because Vercel's servers use anycast networking, it can handle CNAME records differently, allowing for quicker DNS resolution and therefore a faster website experience for the end user.
###  [Subdomain](https://vercel.com/docs/multi-tenant#subdomain)[](https://vercel.com/docs/multi-tenant#subdomain)
A subdomain is a more specific part of that domain that can be assigned to a particular part of your site, for example, `blog.acme.com`, `help.acme.com`. This helps to blend both your brand, with the specificity of where the user may need to go. To add a subdomain to your Project, follow the instructions in the [Add a custom domain](https://vercel.com/docs/domains/add-a-domain#subdomains) doc. If you have bought the domain through Vercel, you can also [point a subdomain to an external service](https://vercel.com/kb/guide/pointing-subdomains-to-external-services) through the Domains section of the dashboard. Subdomains are set through a _CNAME_ DNS record.

###  [Wildcard domain](https://vercel.com/docs/multi-tenant#wildcard-domain)[](https://vercel.com/docs/multi-tenant#wildcard-domain)
You can also configure wildcard domains. Using a wildcard domain, such as `*.acme.com`, is a way to scale and customize your project on Vercel. Rather than specifying a particular subdomain, you can add a wildcard domain to your project, and then you need to set the nameservers to the intended nameservers, allowing the domain to be resolved. See our [multi-tenant SaaS template](https://vercel.com/templates/next.js/platforms-starter-kit) for an example of using wildcard domains on Vercel.
To add a wildcard domain, follow the steps in [Adding a domain](https://vercel.com/docs/domains/add-a-domain#using-wildcard-domain).
Wildcard domains must be configured with the [nameservers method](https://vercel.com/docs/domains/add-a-domain#vercel-nameservers). This is because in order to generate the wildcard certificates, Vercel needs to be able to set DNS records, since the service that Vercel uses to generate those requires us to solve a challenge to verify ownership.
