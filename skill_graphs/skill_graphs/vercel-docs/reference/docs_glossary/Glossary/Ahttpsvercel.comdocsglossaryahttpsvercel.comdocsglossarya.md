##  [A](https://vercel.com/docs/glossary#a)[](https://vercel.com/docs/glossary#a)
###  [Active CPU](https://vercel.com/docs/glossary#active-cpu)[](https://vercel.com/docs/glossary#active-cpu)
A pricing model for [Fluid Compute](https://vercel.com/docs/fluid-compute) where you only pay for the actual CPU time your functions use while executing, rather than provisioned capacity.
###  [AI Gateway](https://vercel.com/docs/glossary#ai-gateway)[](https://vercel.com/docs/glossary#ai-gateway)
A proxy service from Vercel that routes model requests to various AI providers, offering a unified API, budget management, usage monitoring, load balancing, and fallback capabilities.
###  [AI SDK](https://vercel.com/docs/glossary#ai-sdk)[](https://vercel.com/docs/glossary#ai-sdk)
A TypeScript toolkit designed to help developers build AI-powered applications with React, Next.js, Vue, Svelte, and Node.js by providing unified APIs for multiple LLM providers.
###  [Analytics](https://vercel.com/docs/glossary#analytics)[](https://vercel.com/docs/glossary#analytics)
See [Web Analytics](https://vercel.com/docs/glossary#web-analytics).
###  [Anycast Network](https://vercel.com/docs/glossary#anycast-network)[](https://vercel.com/docs/glossary#anycast-network)
A network topology that shares an IP address among multiple nodes, routing requests to the nearest available node based on network conditions to improve performance and fault tolerance.
