##  [M](https://vercel.com/docs/glossary#m)[](https://vercel.com/docs/glossary#m)
###  [Managed Infrastructure](https://vercel.com/docs/glossary#managed-infrastructure)[](https://vercel.com/docs/glossary#managed-infrastructure)
Vercel's fully managed platform that handles server provisioning, scaling, security, and maintenance automatically.
###  [MCP (Model Context Protocol)](https://vercel.com/docs/glossary#mcp-model-context-protocol)[](https://vercel.com/docs/glossary#mcp-model-context-protocol)
A protocol for AI applications that enables secure and standardized communication between AI models and external data sources.
###  [Middleware](https://vercel.com/docs/glossary#middleware)[](https://vercel.com/docs/glossary#middleware)
See [Routing Middleware](https://vercel.com/docs/glossary#routing-middleware).
###  [Microfrontends](https://vercel.com/docs/glossary#microfrontends)[](https://vercel.com/docs/glossary#microfrontends)
A development approach that allows you to split a single application into smaller, independently deployable units that render as one cohesive application for users. Different teams can use different technologies to develop, test, and deploy each microfrontend independently.
###  [Monorepo](https://vercel.com/docs/glossary#monorepo)[](https://vercel.com/docs/glossary#monorepo)
A version control strategy where multiple packages or modules are stored in a single repository, facilitating code sharing and collaboration.
###  [Multi-repo](https://vercel.com/docs/glossary#multi-repo)[](https://vercel.com/docs/glossary#multi-repo)
A version control strategy where each package or module has its own separate repository, also known as "polyrepo."
###  [Multi-tenant](https://vercel.com/docs/glossary#multi-tenant)[](https://vercel.com/docs/glossary#multi-tenant)
Applications that serve multiple customers (tenants) from a single codebase, with each tenant getting their own domain or subdomain.
