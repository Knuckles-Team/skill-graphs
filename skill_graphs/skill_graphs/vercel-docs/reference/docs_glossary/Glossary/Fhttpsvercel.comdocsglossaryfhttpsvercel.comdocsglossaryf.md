##  [F](https://vercel.com/docs/glossary#f)[](https://vercel.com/docs/glossary#f)
###  [Fast Data Transfer](https://vercel.com/docs/glossary#fast-data-transfer)[](https://vercel.com/docs/glossary#fast-data-transfer)
Data transfer between the Vercel CDN and user devices, optimized for performance and charged based on usage.
###  [Feature Flags](https://vercel.com/docs/glossary#feature-flags)[](https://vercel.com/docs/glossary#feature-flags)
Configuration switches that allow you to enable or disable features without deploying new code. Vercel offers the [Flags SDK](https://vercel.com/docs/feature-flags) for implementing flags in Next.js and SvelteKit, and Flags Explorer for managing flags through the Vercel Toolbar.
###  [Firewall](https://vercel.com/docs/glossary#firewall)[](https://vercel.com/docs/glossary#firewall)
See [Vercel Firewall](https://vercel.com/docs/glossary#vercel-firewall).
###  [Fluid Compute](https://vercel.com/docs/glossary#fluid-compute)[](https://vercel.com/docs/glossary#fluid-compute)
An enhanced execution model for Vercel Functions that provides in-function concurrency, and a new pricing model where you only pay for the actual CPU time your functions use while executing, rather than provisioned capacity.
###  [Framework](https://vercel.com/docs/glossary#framework)[](https://vercel.com/docs/glossary#framework)
A software library that provides a foundation for building applications. Vercel supports over 30 frameworks including Next.js, React, Vue, and Svelte.
###  [Framework Preset](https://vercel.com/docs/glossary#framework-preset)[](https://vercel.com/docs/glossary#framework-preset)
A configuration setting that tells Vercel which framework your project uses, enabling automatic optimization and build configuration.
###  [Functions](https://vercel.com/docs/glossary#functions)[](https://vercel.com/docs/glossary#functions)
See [Vercel Functions](https://vercel.com/docs/glossary#vercel-functions).
