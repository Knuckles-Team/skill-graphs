##  [R](https://vercel.com/docs/glossary#r)[](https://vercel.com/docs/glossary#r)
###  [Real Experience Score (RES)](https://vercel.com/docs/glossary#real-experience-score-res)[](https://vercel.com/docs/glossary#real-experience-score-res)
A performance metric in Speed Insights that uses real user data to measure your application's actual performance in production.
###  [Redirects](https://vercel.com/docs/glossary#redirects)[](https://vercel.com/docs/glossary#redirects)
HTTP responses that tell clients to make a new request to a different URL, useful for enforcing HTTPS or directing traffic.
###  [Region](https://vercel.com/docs/glossary#region)[](https://vercel.com/docs/glossary#region)
Geographic locations where Vercel can run your functions and store data. Vercel has 20 compute-capable regions globally.
###  [Repository](https://vercel.com/docs/glossary#repository)[](https://vercel.com/docs/glossary#repository)
A location where files and source code are stored and managed in version control systems like Git, maintaining history of all changes.
###  [Rewrites](https://vercel.com/docs/glossary#rewrites)[](https://vercel.com/docs/glossary#rewrites)
URL transformations that change what the server fetches internally without changing the URL visible to the client.
###  [Routing Middleware](https://vercel.com/docs/glossary#routing-middleware)[](https://vercel.com/docs/glossary#routing-middleware)
Code that executes before a request is processed, running on [Fluid Compute](https://vercel.com/docs/fluid-compute) to modify responses, implement authentication, or perform redirects.
###  [Runtime](https://vercel.com/docs/glossary#runtime)[](https://vercel.com/docs/glossary#runtime)
The execution environment for your functions, such as Node.js, Edge Runtime, Python, or other supported runtimes.
###  [Runtime Logs](https://vercel.com/docs/glossary#runtime-logs)[](https://vercel.com/docs/glossary#runtime-logs)
Logs generated by your functions during execution, useful for debugging and monitoring application behavior.
