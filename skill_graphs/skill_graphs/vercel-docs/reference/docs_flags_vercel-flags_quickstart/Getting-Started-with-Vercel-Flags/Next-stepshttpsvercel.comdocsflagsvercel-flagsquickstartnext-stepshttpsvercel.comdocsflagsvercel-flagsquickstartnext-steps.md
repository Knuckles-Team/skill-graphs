##  [Next steps](https://vercel.com/docs/flags/vercel-flags/quickstart#next-steps)[](https://vercel.com/docs/flags/vercel-flags/quickstart#next-steps)
Your flag is working. Here's what to explore next:
  * [Entities and targeting](https://vercel.com/docs/flags/vercel-flags/dashboard/entities): Define user attributes and create rules to show flags to specific groups.
  * [Segments](https://vercel.com/docs/flags/vercel-flags/dashboard/segments): Build reusable audience groups like "Beta Testers" or "Internal Team."
  * [Flags Explorer](https://vercel.com/docs/flags/flags-explorer/getting-started): Override flags in the Vercel Toolbar during development without affecting other users.
  * [Drafts](https://vercel.com/docs/flags/vercel-flags/dashboard/drafts): Define flags in code first, then promote them in the dashboard when you're ready.
  * [Observability](https://vercel.com/docs/flags/observability): Track flag evaluations in Runtime Logs and Web Analytics.
  * [Managing flags](https://vercel.com/docs/flags/vercel-flags/dashboard): Configure rules, environments, and flag lifecycles in the dashboard.
