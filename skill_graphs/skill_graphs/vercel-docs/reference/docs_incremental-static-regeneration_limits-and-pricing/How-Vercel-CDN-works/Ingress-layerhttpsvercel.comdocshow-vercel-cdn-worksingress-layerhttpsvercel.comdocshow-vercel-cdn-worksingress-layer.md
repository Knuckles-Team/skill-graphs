##  [Ingress layer](https://vercel.com/docs/how-vercel-cdn-works#ingress-layer)[](https://vercel.com/docs/how-vercel-cdn-works#ingress-layer)
Requests arrive at the nearest [Point of Presence (PoP)](https://vercel.com/docs/regions#points-of-presence-pops) out of 126 locations worldwide. The PoP terminates TCP and routes the request to the nearest [Vercel region](https://vercel.com/docs/regions) over a private network in single-digit milliseconds. The region terminates TLS, applies [compression](https://vercel.com/docs/how-vercel-cdn-works/compression), and processes the request through the remaining layers.
