# Projects overview
Last updated December 4, 2025
Projects on Vercel represent applications that you have deployed to the platform from a [single Git repository](https://vercel.com/docs/git). Each project can have multiple deployments: a single production deployment and many pre-production deployments. A project groups [deployments](https://vercel.com/docs/deployments) and [custom domains](https://vercel.com/docs/domains/add-a-domain).
While each project is only connected to a single, imported Git repository, you can have multiple projects connected to a single Git repository that includes many directories, which is particularly useful for [monorepo](https://vercel.com/docs/monorepos) setups.
You can view all projects in your team's [Vercel dashboard](https://vercel.com/dashboard), and selecting a project opens that project's dashboard, where you can:
  * View an overview of the [production deployment](https://vercel.com/docs/deployments) and any active pre-production deployments.
  * Configure [project settings](https://vercel.com/docs/project-configuration/project-settings) such as setting [custom domains](https://vercel.com/docs/domains), [environment variables](https://vercel.com/docs/environment-variables), [deployment protection](https://vercel.com/docs/security/deployment-protection), and more.
  * View details about each [deployment](https://vercel.com/docs/deployments) for that project, such as the status, the commit that triggered the deployment, the deployment URL, and more.
  * Manage [observability](https://vercel.com/docs/observability) for that project, including [Web Analytics](https://vercel.com/docs/analytics), [Speed Insights](https://vercel.com/docs/speed-insights), and [Logs](https://vercel.com/docs/observability/logs).
  * Managing the project's [firewall](https://vercel.com/docs/vercel-firewall).
