## v1.4 (2020-01-24)[¶](https://docs.pydantic.dev/latest/changelog/#v14-2020-01-24)
  * **Breaking Change:** alias precedence logic changed so aliases on a field always take priority over an alias from `alias_generator` to avoid buggy/unexpected behaviour, see [here](https://docs.pydantic.dev/usage/model_config/#alias-precedence) for details,
  * Add support for unicode and punycode in TLDs,
  * Fix `cls` argument in validators during assignment,
  * completing Luhn algorithm for `PaymentCardNumber`,
  * add support for generics that implement `__get_validators__` like a custom data type,
  * add support for infinite generators with `Iterable`,
  * fix `url_regex` to accept schemas with `+`, `-` and `.` after the first character,
  * move `version_info()` to `version.py`, suggest its use in issues,
  * Improve pydantic import time by roughly 50% by deferring some module loading and regex compilation,
  * Fix `EmailStr` and `NameEmail` to accept instances of themselves in cython,
  * Pass model class to the `Config.schema_extra` callable,
  * Fix regex for username and password in URLs,
  * Add support for nested generic models,
  * add `__all__` to `__init__.py` to prevent "implicit reexport" errors from mypy,
  * Add support for using "dotenv" files with `BaseSettings`,
