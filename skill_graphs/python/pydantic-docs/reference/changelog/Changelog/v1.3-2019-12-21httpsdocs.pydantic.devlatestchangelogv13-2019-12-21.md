## v1.3 (2019-12-21)[¶](https://docs.pydantic.dev/latest/changelog/#v13-2019-12-21)
  * Change `schema` and `schema_model` to handle dataclasses by using their `__pydantic_model__` feature,
  * Added option for `root_validator` to be skipped if values validation fails using keyword `skip_on_failure=True`,
  * Allow `Config.schema_extra` to be a callable so that the generated schema can be post-processed,
  * Update mypy to version 0.750,
  * Trick Cython into allowing str subclassing,
  * Prevent type attributes being added to schema unless the attribute `__schema_attributes__` is `True`,
  * Change `BaseModel.parse_file` to use `Config.json_loads`,
  * Fix for optional `Json` fields,
  * Change the default number of threads used when compiling with cython to one, allow override via the `CYTHON_NTHREADS` environment variable,
  * Run FastAPI tests during Pydantic's CI tests,
  * My mypy strictness constraints, and associated tweaks to type annotations,
  * Add `__eq__` to SecretStr and SecretBytes to allow "value equals",
  * Fix schema generation for nested None case,
  * Consistent checks for sequence like objects,
  * Fix `Config` inheritance on `BaseSettings` when used with `env_prefix`,
  * Fix for `__modify_schema__` when it conflicted with `field_class_to_schema*`,
  * docs: Fix explanation of case sensitive environment variable names when populating `BaseSettings` subclass attributes,
  * Rename django-rest-framework benchmark in documentation,
