### Changes[Â¶](https://docs.pydantic.dev/latest/changelog/#changes_14)
  * **Breaking Change:** remove `__field_defaults__`, add `default_factory` support with `BaseModel.construct`. Use `.get_default()` method on fields in `__fields__` attribute instead,
  * Rearrange CI to run linting as a separate job, split install recipes for different tasks,
  * Allows subclasses of generic models to make some, or all, of the superclass's type parameters concrete, while also defining new type parameters in the subclass,
  * Call validator with the correct `values` parameter type in `BaseModel.__setattr__`, when `validate_assignment = True` in model config,
  * Force `fields.Undefined` to be a singleton object, fixing inherited generic model schemas,
  * Include tests in source distributions,
  * Add ability to use `min_length/max_length` constraints with secret types,
  * Also check `root_validators` when `validate_assignment` is on,
  * Fix const validators not running when custom validators are present,
  * add `deque` to field types,
  * add basic support for Python 3.9,
  * Fix typo in the anchor of exporting_models.md#modelcopy and incorrect description,
  * Added ability for `BaseSettings` to read "secret files",
  * add `parse_raw_as` utility function,
  * Support home directory relative paths for `dotenv` files (e.g. `~/.env`),
  * Clarify documentation for `parse_file` to show that the argument should be a file _path_ not a file-like object,
  * Fix false positive from mypy plugin when a class nested within a `BaseModel` is named `Model`,
  * add basic support of Pattern type in schema generation,
  * Support custom title, description and default in schema of enums,
  * Properly represent `Literal` Enums when `use_enum_values` is True,
  * Allows timezone information to be added to strings to be formatted as time objects. Permitted formats are `Z` for UTC or an offset for absolute positive or negative time shifts. Or the timezone data can be omitted,
  * Add stub `__init__` with Python 3.6 signature for `ForwardRef`,
  * Fix behaviour with forward refs and optional fields in nested models,
  * add `Enum` and `IntEnum` as valid types for fields,
  * Change default value of `__module__` argument of `create_model` from `None` to `'pydantic.main'`. Set reference of created concrete model to it's module to allow pickling (not applied to models created in functions),
  * Add private attributes support,
  * add `config` to `@validate_arguments`,
  * Allow descendant Settings models to override env variable names for the fields defined in parent Settings models with `env` in their `Config`. Previously only `env_prefix` configuration option was applicable,
  * Support `ref_template` when creating schema `$ref`s,
  * Add a `__call__` stub to `PyObject` so that mypy will know that it is callable,
  * `pydantic.dataclasses.dataclass` decorator now supports built-in `dataclasses.dataclass`. It is hence possible to convert an existing `dataclass` easily to add Pydantic validation. Moreover nested dataclasses are also supported,
