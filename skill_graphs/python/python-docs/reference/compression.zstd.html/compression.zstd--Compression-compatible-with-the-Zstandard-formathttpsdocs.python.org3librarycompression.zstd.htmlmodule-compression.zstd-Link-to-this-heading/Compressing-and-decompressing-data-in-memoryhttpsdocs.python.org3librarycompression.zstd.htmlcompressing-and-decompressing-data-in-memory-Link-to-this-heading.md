## Compressing and decompressing data in memory[¶](https://docs.python.org/3/library/compression.zstd.html#compressing-and-decompressing-data-in-memory "Link to this heading")

compression.zstd.compress(_data_ , _level =None_, _options =None_, _zstd_dict =None_)[¶](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.compress "Link to this definition")

Compress _data_ (a [bytes-like object](https://docs.python.org/3/glossary.html#term-bytes-like-object)), returning the compressed data as a [`bytes`](https://docs.python.org/3/library/stdtypes.html#bytes "bytes") object.
The _level_ argument is an integer controlling the level of compression. _level_ is an alternative to setting [`CompressionParameter.compression_level`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.CompressionParameter.compression_level "compression.zstd.CompressionParameter.compression_level") in _options_. Use [`bounds()`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.CompressionParameter.bounds "compression.zstd.CompressionParameter.bounds") on `compression_level` to get the values that can be passed for _level_. If advanced compression options are needed, the _level_ argument must be omitted and in the _options_ dictionary the `CompressionParameter.compression_level` parameter should be set.
The _options_ argument is a Python dictionary containing advanced compression parameters. The valid keys and values for compression parameters are documented as part of the [`CompressionParameter`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.CompressionParameter "compression.zstd.CompressionParameter") documentation.
The _zstd_dict_ argument is an instance of [`ZstdDict`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdDict "compression.zstd.ZstdDict") containing trained data to improve compression efficiency. The function [`train_dict()`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.train_dict "compression.zstd.train_dict") can be used to generate a Zstandard dictionary.

compression.zstd.decompress(_data_ , _zstd_dict =None_, _options =None_)[¶](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.decompress "Link to this definition")

Decompress _data_ (a [bytes-like object](https://docs.python.org/3/glossary.html#term-bytes-like-object)), returning the uncompressed data as a [`bytes`](https://docs.python.org/3/library/stdtypes.html#bytes "bytes") object.
The _options_ argument is a Python dictionary containing advanced decompression parameters. The valid keys and values for compression parameters are documented as part of the [`DecompressionParameter`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.DecompressionParameter "compression.zstd.DecompressionParameter") documentation.
The _zstd_dict_ argument is an instance of [`ZstdDict`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdDict "compression.zstd.ZstdDict") containing trained data used during compression. This must be the same Zstandard dictionary used during compression.
If _data_ is the concatenation of multiple distinct compressed frames, decompress all of these frames, and return the concatenation of the results.

_class_ compression.zstd.ZstdCompressor(_level =None_, _options =None_, _zstd_dict =None_)[¶](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdCompressor "Link to this definition")

Create a compressor object, which can be used to compress data incrementally.
For a more convenient way of compressing a single chunk of data, see the module-level function [`compress()`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.compress "compression.zstd.compress").
The _level_ argument is an integer controlling the level of compression. _level_ is an alternative to setting [`CompressionParameter.compression_level`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.CompressionParameter.compression_level "compression.zstd.CompressionParameter.compression_level") in _options_. Use [`bounds()`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.CompressionParameter.bounds "compression.zstd.CompressionParameter.bounds") on `compression_level` to get the values that can be passed for _level_. If advanced compression options are needed, the _level_ argument must be omitted and in the _options_ dictionary the `CompressionParameter.compression_level` parameter should be set.
The _options_ argument is a Python dictionary containing advanced compression parameters. The valid keys and values for compression parameters are documented as part of the [`CompressionParameter`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.CompressionParameter "compression.zstd.CompressionParameter") documentation.
The _zstd_dict_ argument is an optional instance of [`ZstdDict`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdDict "compression.zstd.ZstdDict") containing trained data to improve compression efficiency. The function [`train_dict()`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.train_dict "compression.zstd.train_dict") can be used to generate a Zstandard dictionary.

compress(_data_ , _mode =ZstdCompressor.CONTINUE_)[¶](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdCompressor.compress "Link to this definition")

Compress _data_ (a [bytes-like object](https://docs.python.org/3/glossary.html#term-bytes-like-object)), returning a [`bytes`](https://docs.python.org/3/library/stdtypes.html#bytes "bytes") object with compressed data if possible, or otherwise an empty `bytes` object. Some of _data_ may be buffered internally, for use in later calls to `compress()` and [`flush()`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdCompressor.flush "compression.zstd.ZstdCompressor.flush"). The returned data should be concatenated with the output of any previous calls to `compress()`.
The _mode_ argument is a `ZstdCompressor` attribute, either [`CONTINUE`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdCompressor.CONTINUE "compression.zstd.ZstdCompressor.CONTINUE"), [`FLUSH_BLOCK`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdCompressor.FLUSH_BLOCK "compression.zstd.ZstdCompressor.FLUSH_BLOCK"), or [`FLUSH_FRAME`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdCompressor.FLUSH_FRAME "compression.zstd.ZstdCompressor.FLUSH_FRAME").
When all data has been provided to the compressor, call the [`flush()`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdCompressor.flush "compression.zstd.ZstdCompressor.flush") method to finish the compression process. If `compress()` is called with _mode_ set to [`FLUSH_FRAME`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdCompressor.FLUSH_FRAME "compression.zstd.ZstdCompressor.FLUSH_FRAME"), `flush()` should not be called, as it would write out a new empty frame.

flush(_mode =ZstdCompressor.FLUSH_FRAME_)[¶](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdCompressor.flush "Link to this definition")

Finish the compression process, returning a [`bytes`](https://docs.python.org/3/library/stdtypes.html#bytes "bytes") object containing any data stored in the compressor’s internal buffers.
The _mode_ argument is a `ZstdCompressor` attribute, either [`FLUSH_BLOCK`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdCompressor.FLUSH_BLOCK "compression.zstd.ZstdCompressor.FLUSH_BLOCK"), or [`FLUSH_FRAME`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdCompressor.FLUSH_FRAME "compression.zstd.ZstdCompressor.FLUSH_FRAME").

set_pledged_input_size(_size_)[¶](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdCompressor.set_pledged_input_size "Link to this definition")

Specify the amount of uncompressed data _size_ that will be provided for the next frame. _size_ will be written into the frame header of the next frame unless [`CompressionParameter.content_size_flag`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.CompressionParameter.content_size_flag "compression.zstd.CompressionParameter.content_size_flag") is `False` or `0`. A size of `0` means that the frame is empty. If _size_ is `None`, the frame header will omit the frame size. Frames that include the uncompressed data size require less memory to decompress, especially at higher compression levels.
If [`last_mode`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdCompressor.last_mode "compression.zstd.ZstdCompressor.last_mode") is not [`FLUSH_FRAME`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdCompressor.FLUSH_FRAME "compression.zstd.ZstdCompressor.FLUSH_FRAME"), a [`ValueError`](https://docs.python.org/3/library/exceptions.html#ValueError "ValueError") is raised as the compressor is not at the start of a frame. If the pledged size does not match the actual size of data provided to [`compress()`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdCompressor.compress "compression.zstd.ZstdCompressor.compress"), future calls to `compress()` or [`flush()`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdCompressor.flush "compression.zstd.ZstdCompressor.flush") may raise [`ZstdError`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdError "compression.zstd.ZstdError") and the last chunk of data may be lost.
After [`flush()`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdCompressor.flush "compression.zstd.ZstdCompressor.flush") or [`compress()`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdCompressor.compress "compression.zstd.ZstdCompressor.compress") are called with mode [`FLUSH_FRAME`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdCompressor.FLUSH_FRAME "compression.zstd.ZstdCompressor.FLUSH_FRAME"), the next frame will not include the frame size into the header unless `set_pledged_input_size()` is called again.

CONTINUE[¶](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdCompressor.CONTINUE "Link to this definition")

Collect more data for compression, which may or may not generate output immediately. This mode optimizes the compression ratio by maximizing the amount of data per block and frame.

FLUSH_BLOCK[¶](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdCompressor.FLUSH_BLOCK "Link to this definition")

Complete and write a block to the data stream. The data returned so far can be immediately decompressed. Past data can still be referenced in future blocks generated by calls to [`compress()`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdCompressor.compress "compression.zstd.ZstdCompressor.compress"), improving compression.

FLUSH_FRAME[¶](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdCompressor.FLUSH_FRAME "Link to this definition")

Complete and write out a frame. Future data provided to [`compress()`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdCompressor.compress "compression.zstd.ZstdCompressor.compress") will be written into a new frame and _cannot_ reference past data.

last_mode[¶](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdCompressor.last_mode "Link to this definition")

The last mode passed to either [`compress()`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdCompressor.compress "compression.zstd.ZstdCompressor.compress") or [`flush()`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdCompressor.flush "compression.zstd.ZstdCompressor.flush"). The value can be one of [`CONTINUE`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdCompressor.CONTINUE "compression.zstd.ZstdCompressor.CONTINUE"), [`FLUSH_BLOCK`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdCompressor.FLUSH_BLOCK "compression.zstd.ZstdCompressor.FLUSH_BLOCK"), or [`FLUSH_FRAME`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdCompressor.FLUSH_FRAME "compression.zstd.ZstdCompressor.FLUSH_FRAME"). The initial value is `FLUSH_FRAME`, signifying that the compressor is at the start of a new frame.

_class_ compression.zstd.ZstdDecompressor(_zstd_dict =None_, _options =None_)[¶](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdDecompressor "Link to this definition")

Create a decompressor object, which can be used to decompress data incrementally.
For a more convenient way of decompressing an entire compressed stream at once, see the module-level function [`decompress()`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.decompress "compression.zstd.decompress").
The _options_ argument is a Python dictionary containing advanced decompression parameters. The valid keys and values for compression parameters are documented as part of the [`DecompressionParameter`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.DecompressionParameter "compression.zstd.DecompressionParameter") documentation.
The _zstd_dict_ argument is an instance of [`ZstdDict`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdDict "compression.zstd.ZstdDict") containing trained data used during compression. This must be the same Zstandard dictionary used during compression.
Note
This class does not transparently handle inputs containing multiple compressed frames, unlike the [`decompress()`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.decompress "compression.zstd.decompress") function and [`ZstdFile`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdFile "compression.zstd.ZstdFile") class. To decompress a multi-frame input, you should use `decompress()`, `ZstdFile` if working with a [file object](https://docs.python.org/3/glossary.html#term-file-object), or multiple `ZstdDecompressor` instances.

decompress(_data_ , _max_length =-1_)[¶](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdDecompressor.decompress "Link to this definition")

Decompress _data_ (a [bytes-like object](https://docs.python.org/3/glossary.html#term-bytes-like-object)), returning uncompressed data as bytes. Some of _data_ may be buffered internally, for use in later calls to `decompress()`. The returned data should be concatenated with the output of any previous calls to `decompress()`.
If _max_length_ is non-negative, the method returns at most _max_length_ bytes of decompressed data. If this limit is reached and further output can be produced, the [`needs_input`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdDecompressor.needs_input "compression.zstd.ZstdDecompressor.needs_input") attribute will be set to `False`. In this case, the next call to `decompress()` may provide _data_ as `b''` to obtain more of the output.
If all of the input data was decompressed and returned (either because this was less than _max_length_ bytes, or because _max_length_ was negative), the [`needs_input`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdDecompressor.needs_input "compression.zstd.ZstdDecompressor.needs_input") attribute will be set to `True`.
Attempting to decompress data after the end of a frame will raise a [`ZstdError`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdError "compression.zstd.ZstdError"). Any data found after the end of the frame is ignored and saved in the [`unused_data`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdDecompressor.unused_data "compression.zstd.ZstdDecompressor.unused_data") attribute.

eof[¶](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdDecompressor.eof "Link to this definition")

`True` if the end-of-stream marker has been reached.

unused_data[¶](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdDecompressor.unused_data "Link to this definition")

Data found after the end of the compressed stream.
Before the end of the stream is reached, this will be `b''`.

needs_input[¶](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdDecompressor.needs_input "Link to this definition")

`False` if the [`decompress()`](https://docs.python.org/3/library/compression.zstd.html#compression.zstd.ZstdDecompressor.decompress "compression.zstd.ZstdDecompressor.decompress") method can provide more decompressed data before requiring new compressed input.
