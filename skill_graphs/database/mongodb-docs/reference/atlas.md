# What is MongoDB Atlas?[](https://www.mongodb.com/docs/atlas/#what-is-service-fullname- "Permalink to this heading")
MongoDB Atlas is a multi-cloud database service by the same people that build MongoDB. Atlas simplifies deploying and managing your databases while offering the versatility you need to build resilient and performant global applications on the cloud providers of your choice.
[Get Started](https://www.mongodb.com/docs/atlas/getting-started/)
![MongoDB Atlas landing page hero image](https://www.mongodb.com/docs/atlas/images/atlas-plp/atlas-plp-hero.svg)
What You Can Do
## Build with MongoDB on AWS, Azure, and Google Cloud[](https://www.mongodb.com/docs/atlas/#build-with-mongodb-on-aws--azure--and-google-cloud "Permalink to this heading")
MongoDB Atlas makes it easy to deploy and manage databases on-demand when and where you need them.
Deploy a Database
Secure your Database
Connect to your Database
Optimize your Database
1
### Choose a cluster type.[](https://www.mongodb.com/docs/atlas/#choose-a-cluster-type. "Permalink to this heading")
Play around with a free cluster, launch a Flex cluster, or define a dedicated cluster configuration for your application.
To choose a deployment type, see [Database Deployment Types.](https://www.mongodb.com/docs/atlas/create-database-deployment/#std-label-ref-deployment-types)
2
### Choose a Cloud Provider and Region[](https://www.mongodb.com/docs/atlas/#choose-a-cloud-provider-and-region "Permalink to this heading")
Deploy your database to the same cloud provider and region as your applications to reduce latency and standardize security controls.
To choose a cloud provider and region, see [Cloud Providers and Regions.](https://www.mongodb.com/docs/atlas/cloud-providers-regions/#std-label-create-cluster-cloud-provider-region)
3
### Customize your cluster.[](https://www.mongodb.com/docs/atlas/#customize-your-cluster. "Permalink to this heading")
Enable multi-cloud and multi-region data distribution to expand global coverage, increase fault tolerance, and meet data compliance requirements.
To customize your cluster, see [Configure High Availability and Workload Isolation.](https://www.mongodb.com/docs/atlas/cluster-config/multi-cloud-distribution/#std-label-create-cluster-multi-region)
![MongoDB Atlas Deploy a Database image](https://www.mongodb.com/docs/atlas/images/atlas-plp/atlas-plp-deploy-db.svg)
1
### Add IP Access List Entries[](https://www.mongodb.com/docs/atlas/#add-ip-access-list-entries "Permalink to this heading")
Define an IP access list for your cluster. Always-on authentication ensures only approved client connections can access your database.
To add IP Access List entries, see [Configure IP Access List Entries.](https://www.mongodb.com/docs/atlas/security/ip-access-list/#std-label-access-list)
2
### Manage Database Users[](https://www.mongodb.com/docs/atlas/#manage-database-users "Permalink to this heading")
Define how your team members and applications authenticate to your database and what data they can access.
To manage database users, see [Configure Database Users.](https://www.mongodb.com/docs/atlas/security-add-mongodb-users/#std-label-mongodb-users)
3
### (Optional) Configure Private Network Access[](https://www.mongodb.com/docs/atlas/#-optional--configure-private-network-access "Permalink to this heading")
Configure peering connections or private endpoints for your applications to connect to your databases for additional security controls.
To configure peering connections, see [Set Up a Network Peering Connection](https://www.mongodb.com/docs/atlas/security-vpc-peering/#std-label-vpc-peering). To configure private endpoints, see [Configure Private Endpoints.](https://www.mongodb.com/docs/atlas/security-configure-private-endpoints/#std-label-private-endpoint)
![MongoDB Atlas Secure your Database image](https://www.mongodb.com/docs/atlas/images/atlas-plp/atlas-plp-secure-db.svg)
1
### Choose a Connection Type[](https://www.mongodb.com/docs/atlas/#choose-a-connection-type "Permalink to this heading")
Connect to your database using the MongoDB Shell, one of MongoDB's client libraries, MongoDB Compass, or the MongoDB Connector for BI.
To connect to your database, see [Connect to an Atlas Cluster.](https://www.mongodb.com/docs/atlas/connect-to-database-deployment/#std-label-atlas-connect-to-deployment)
2
### Interact with your Data[](https://www.mongodb.com/docs/atlas/#interact-with-your-data "Permalink to this heading")
Use your chosen connection type to view your data, import documents, and run queries.
To insert data, see [Insert and View a Document.](https://www.mongodb.com/docs/atlas/tutorial/insert-data-into-your-cluster/#std-label-gswa-insert-data)
![MongoDB Atlas Connect to your Database image](https://www.mongodb.com/docs/atlas/images/atlas-plp/atlas_plp.gif)
1
### Create Custom Alerts[](https://www.mongodb.com/docs/atlas/#create-custom-alerts "Permalink to this heading")
Customize default alerts or create new ones to receive notifications based on events and metrics that you define.
To create custom alerts, see [Configure and Resolve Alerts.](https://www.mongodb.com/docs/atlas/alerts/#std-label-alerts)
2
### Review Index and Schema Suggestions[](https://www.mongodb.com/docs/atlas/#review-index-and-schema-suggestions "Permalink to this heading")
See on-demand index and schema suggestions for your collections to improve your application's query performance.
To review suggestions, see [Analyze Slow Queries](https://www.mongodb.com/docs/atlas/analyze-slow-queries/#std-label-analyze-slow-queries) and [Improve Your Schema.](https://www.mongodb.com/docs/atlas/performance-advisor/schema-suggestions/#std-label-schema-suggestions)
3
### Automate Archival of Cold Data[](https://www.mongodb.com/docs/atlas/#automate-archival-of-cold-data "Permalink to this heading")
Define archival rules to move infrequently accessed data into fully managed cloud object storage while retaining full queryability of that data.
To archive your data, see [Online Archive Overview.](https://www.mongodb.com/docs/atlas/online-archive/overview/#std-label-online-archive-overview)
